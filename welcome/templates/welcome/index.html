Models Examples

This example includes three applications:
  • todolist2 -- a version of the shared To Do List that stores its data in a database using a Django model.
  • addrbook2 -- a version of the Address Book example that stores is data in the database, using a Django model, and uses ModelForms to reduce duplication of code.
  • sample -- a "sample" model, which you cannot run, but we will use to demonstrate accessing models from the python shell.
Note: You will be asked to register/login to use these examples.
Note: These examples store data in the database. The settings.py file is configured to store the data in SQLite. The first time you run these examples will need to makemigrations and then migrate to set up the database before runserver. If you change the models, don't forget to makemigrations and migrate again.
